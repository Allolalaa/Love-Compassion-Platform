## 🔧 المرحلة الثانية: نظام إدارة المحتوى (CMS)

**المدة المتوقعة**: 3-4 أشهر

### الهدف:
تمكين المتطوعين غير التقنيين من إدارة المحتوى بالكامل بدون الحاجة لتعديل الكود.

### الميزات المخطط لها:

#### 1. لوحة التحكم (Dashboard)
```
📊 نظرة عامة على:
- إجمالي الأعضاء
- الفيديوهات والدورات
- إحصائيات الزيارات
- الفعاليات القادمة
```

#### 2. إدارة الفيديو
```
✨ يمكن للمسؤول:
- رفع فيديو من الجهاز أو رابط خارجي
- إضافة عنوان ووصف
- اختيار التصنيف من قائمة منسدلة
- رفع صورة مصغرة
- نشر أو جدولة النشر
- تعديل أو حذف الفيديو
```

#### 3. إدارة الرسوم البيانية
```
✨ إمكانيات:
- رفع صور الرسوم البيانية
- إضافة العنوان والوصف
- تصنيف تلقائي بالذكاء الاصطناعي
- معاينة قبل النشر
```

#### 4. إدارة الدورات
```
✨ منشئ الدورات:
- واجهة سحب وإفلات لترتيب الدروس
- إضافة محاور ووحدات
- رفع مواد تعليمية (PDF, فيديو)
- تحديد المستوى والمدة
- إضافة اختبارات تقييمية
```

#### 5. إدارة المنتدى
```
✨ أدوات الإشراف:
- الموافقة على المواضيع الجديدة
- تثبيت/إلغاء تثبيت المواضيع
- حذف أو تعديل الردود
- حظر المستخدمين المخالفين
- إحصائيات النشاط
```

#### 6. إدارة الفعاليات
```
✨ منظم الفعاليات:
- إنشاء فعالية بنموذج سهل
- تكامل مع Google Calendar
- إرسال دعوات تلقائية
- إدارة قائمة المسجلين
- رفع التسجيل تلقائياً بعد الانتهاء
```

### التقنيات المقترحة:
- **Backend**: Node.js + Express أو Firebase
- **Database**: MongoDB أو PostgreSQL
- **File Storage**: AWS S3 أو Cloudinary
- **Admin Panel**: React Admin أو Strapi
