# ⚡ دليل البدء السريع

## 🎯 للمتطوعين غير التقنيين

هذا دليل مبسط جداً لإدارة المحتوى على المنصة بدون خبرة برمجية.

---

## 📝 قبل البدء

### ما تحتاجه:
1. ✅ محرر نصوص (نوصي بـ Visual Studio Code)
2. ✅ متصفح حديث (Chrome, Firefox, Safari)
3. ✅ الملفات المراد رفعها (صور، روابط فيديوهات)

### تحميل VS Code:
1. اذهب إلى: https://code.visualstudio.com
2. حمل النسخة المناسبة لنظامك
3. ثبت البرنامج

---

## 🎥 إضافة فيديو جديد في 5 خطوات

### الخطوة 1: افتح المشروع
```
1. افتح VS Code
2. اذهب إلى File → Open Folder
3. اختر مجلد المشروع
```

### الخطوة 2: افتح ملف الفيديو
```
في القائمة الجانبية:
src → components → VideoLibrary.tsx
```

### الخطوة 3: ابحث عن مكان الإضافة
```
اضغط Ctrl+F (أو Cmd+F على Mac)
ابحث عن: const videos = [
```

### الخطوة 4: أضف الفيديو
```javascript
// انسخ هذا النموذج وألصقه قبل آخر قوس مربع ]
{
  id: 7,  // غير الرقم
  title: 'ضع عنوان الفيديو هنا',
  category: 'talks',  // اختر: talks, interviews, tutorials, shorts
  thumbnail: 'رابط الصورة المصغرة',
  duration: '15:30',
  views: '0',
  date: 'منذ يوم',
},
```

### الخطوة 5: احفظ
```
اضغط Ctrl+S (أو Cmd+S على Mac)
```

## 🚀 نشر التغييرات

### بعد كل التعديلات:

1. **احفظ كل الملفات**
   ```
   File → Save All
   أو Ctrl+K ثم S
   ```

2. **ابنِ المشروع**
   ```
   افتح Terminal في VS Code
   اكتب: npm run build
   اضغط Enter
   ```

3. **ارفع الملفات**
   ```
   ارفع محتوى مجلد dist
   إلى خادم الاستضافة
   ```

## ⚠️ أخطاء شائعة وحلولها

### ❌ الخطأ: لم يظهر التغيير
✅ **الحل**: تأكد من حفظ الملف (Ctrl+S)

### ❌ الخطأ: رسالة خطأ عند البناء
✅ **الحل**: تحقق من:
- هل هناك فاصلة منسية؟
- هل القوس مغلق؟
- هل رقم ID فريد؟

### ❌ الخطأ: الموقع لا يعمل
✅ **الحل**: ارجع للنسخة الاحتياطية

### ❌ الخطأ: الصورة لا تظهر
✅ **الحل**: تأكد من:
- الرابط صحيح
- الصورة متاحة على الإنترنت

## 🎯 نصائح ذهبية

### 1. ✨ النسخ الاحتياطي
> اعمل نسخة احتياطية قبل أي تعديل كبير

### 2. 🎨 الصور
> استخدم صور عالية الجودة (800x450 على الأقل)

> اجعل العناوين واضحة وجذابة

### 4. 🔢 الأرقام
> استخدم أرقام ID فريدة دائماً

### 5. 💾 الحفظ
> احفظ عملك بانتظام

### 6. 🧪 الاختبار
> اختبر التغييرات قبل النشر
